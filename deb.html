PythonAnywhere Deployment Guide

Step-by-step instructions to deploy AI Solutions on PythonAnywhere — written for beginners.

What you will learn in this guide:

  1. How to push this project folder to GitHub (free online code storage)
  2. How to deploy the site on PythonAnywhere (free web hosting)

If you don't understand any term, just follow the steps exactly as written.

0. Push This Project to GitHub

1 What is GitHub?

GitHub is a website that stores your code online. Think of it like Google Drive for code. You'll upload your project here so PythonAnywhere can download it.

2 Create a GitHub account VS CODE

Open your browser, go to github.com/signup, and create a free account. Verify your email address after signing up.

3 Create a new repository on GitHub VS CODE

A "repository" is like a project folder on GitHub. Follow these steps:

  1. Log in to github.com.
  2. Click the green New button (top-left, next to your profile picture). Or click the + icon in the top-right corner and select New repository.
  3. In the Repository name field, type: aisolutions
  4. Leave it Public (free).
  5. DO NOT check "Add a README file" or "Add .gitignore" — leave everything unchecked.
  6. Click the green Create repository button at the bottom.

After creation, you'll see a page with a URL like https://github.com/yourusername/aisolutions. Keep this page open — you'll need it in step 9.

4 Open VS Code and open this project folder VS CODE

Visual Studio Code (VS Code) is the program where you edit code.

  1. Open VS Code.
  2. Click File (top-left) → Open Folder.
  3. Navigate to C:\Users\DOS\Downloads\AI Solutions and click Select Folder.
  4. You should now see all the project files in the left sidebar.
5 Open the terminal in VS Code VS CODE

The terminal is where you type commands. To open it:

  1. In VS Code, click Terminal in the top menu bar.
  2. Click New Terminal.
  3. A panel will open at the bottom of VS Code. This is where you'll type the commands below.
6 Initialize Git in this folder VS CODE

Type this command in the VS Code terminal and press Enter:

git init

This tells Git (the version control system) to start tracking this folder. You should see a message like Initialized empty Git repository.

7 Add all files to Git staging VS CODE

Type this command and press Enter:

git add .

The dot . means "all files in this folder". This tells Git which files to include in the first upload.

8 Create your first commit VS CODE

A "commit" is like a snapshot of your code. Type:

git commit -m "Initial commit - AI Solutions project"

The text in quotes is a message describing what this snapshot contains.

If Git asks you to set your email or name, run these two commands first (replace the email and name with yours):
git config --global user.email "you@example.com"
git config --global user.name "Your Name"
Then run the git commit command again.
9 Link your local folder to GitHub VS CODE

Go back to your GitHub page in the browser (the one you created in step 3). You'll see a section called "…or push an existing repository from the command line". Copy the first line that looks like:

git remote add origin https://github.com/yourusername/aisolutions.git

Paste it into the VS Code terminal and press Enter. This tells your computer: "the online location for this project is at this URL".

10 Push (upload) your code to GitHub VS CODE

Type this command and press Enter:

git push -u origin master

If that gives an error about "main" instead of "master", try:

git push -u origin main

GitHub will ask you to log in. A browser window may pop up asking you to sign in to GitHub — click Sign in and authorize. If it asks for a username and password in the terminal:

  • Username: your GitHub username
  • Password: use a personal access token (not your regular password). To create one:
    1. Go to github.com/settings/tokens
    2. Click Generate new token (classic).
    3. Give it a name like "VS Code push".
    4. Check the box repo (full control).
    5. Click Generate token at the bottom.
    6. Copy the token (it looks like ghp_xxxxxxxxxxxx).
    7. Paste it as the password in the terminal (the cursor won't move — that's normal). Press Enter.
After this step, your code is now on GitHub! Go to https://github.com/yourusername/aisolutions in your browser to see all your files there.
11 Create a .gitignore file (so venv is not uploaded) VS CODE

The venv folder is very large and should not be uploaded to GitHub. Create a file called .gitignore that tells Git to ignore it.

  1. In VS Code, click File → New File.
  2. Type exactly: .gitignore as the filename and press Enter.
  3. Paste this content into the file: 
    venv/
__pycache__/
*.pyc
.env
db.sqlite
django_out.log
django_err.log
*.sqbpro
  4. Save the file (Ctrl+S). Make sure it's saved in C:\Users\DOS\Downloads\AI Solutions.

Now upload this new file to GitHub:

git add .gitignore
git commit -m "add .gitignore"
git push
Done with GitHub! Your code is now online at https://github.com/yourusername/aisolutions.
Continue below for PythonAnywhere deployment.

PythonAnywhere Deployment

1. Account & Console Setup

1 Sign up / Log in to PythonAnywhere

Go to pythonanywhere.com/login/ and log in with your account. If you don't have one, sign up first (free tier is sufficient).

2 Open a Bash console CONSOLE

From the dashboard, click Consoles in the top navigation bar, then click Bash to open a new Bash console. This is where you'll run all the terminal commands.

You can also open multiple consoles if needed. Each console runs independently.

2. Upload the Project Code

3 Option A — Clone from GitHub (recommended) CONSOLE

In the Bash console, run:

git clone https://github.com/yourusername/aisolutions.git
cd aisolutions

Replace yourusername with your actual GitHub username and repository name.

4 Option B — Upload files manually FILES + CONSOLE

If you don't use Git:

  1. Go to the Files tab in PythonAnywhere.
  2. Navigate to /home/yourusername/.
  3. Click Upload a file and upload your project ZIP.
  4. Open a Bash console and run: 
    unzip aisolutions.zip -d aisolutions
cd aisolutions

3. Set Up Virtual Environment

5 Create a virtual environment CONSOLE

Run the following commands in the Bash console (inside the aisolutions directory):

python3 -m venv venv
source venv/bin/activate

Your prompt should now show (venv) at the beginning, indicating the virtual environment is active.

6 Upgrade pip and install dependencies CONSOLE 
pip install --upgrade pip
pip install -r requirements.txt

This will install Django, DRF, Gunicorn, Whitenoise, and all other required packages.

7 Apply database migrations CONSOLE 
python manage.py migrate

This creates all the database tables (events, registrations, inquiries, feedback, etc.) in the SQLite database file.

8 Collect static files CONSOLE 
python manage.py collectstatic --noinput

This copies all CSS, JS, and image files into the staticfiles/ directory so they can be served in production.

9 Deactivate the virtual environment CONSOLE 
deactivate

You're done with the setup. Now we need to configure the web app.

4. Create the Web App

10 Go to the Web tab WEB

Click Web in the top navigation bar of PythonAnywhere.

11 Add a new web app WEB

Click Add a new web app. A wizard will appear:

  1. On the first screen ("Pick your domain name"), your default domain will be yourusername.pythonanywhere.com. Click Next.
  2. On the second screen ("Select a Python Web framework"), click Manual configuration (do NOT select Django here — the wizard creates a new project; we already have one).
  3. On the third screen ("Select Python version"), choose Python 3.12 and click Next.

Wait a few seconds for the web app to be created.

12 Verify the domain WEB

Your web app page will now show your domain at the top. It should look like:

https://yourusername.pythonanywhere.com

Replace yourusername with your actual PythonAnywhere username.

5. Configure the WSGI File

13 Understand the WSGI file WEB

The WSGI (Web Server Gateway Interface) file tells PythonAnywhere how to run your Django application. It's the bridge between the web server and your Python code. We need to edit it to point to your project and set the correct environment variables.

14 Edit the WSGI file WEB

In the Web tab, scroll down to the Code section. You'll see a link labeled:

/var/www/yourusername_pythonanywhere_com_wsgi.py

Click that link to open the file editor.

15 Replace the contents

Delete all the default content and paste the following. Important: Replace yourusername with your actual PythonAnywhere username in three places (marked below).

import os
import sys

# ──────────────────────────────────────────────
# 1. Add your project to Python's module search path
# ──────────────────────────────────────────────
path = '/home/yourusername/aisolutions'          # ← CHANGE this
if path not in sys.path:
    sys.path.append(path)

# ──────────────────────────────────────────────
# 2. Set environment variables (same as your .env file)
# ──────────────────────────────────────────────
os.environ['DEBUG'] = 'False'
os.environ['SECRET_KEY'] = 'your-strong-secret-key-here'
os.environ['ALLOWED_HOSTS'] = 'yourusername.pythonanywhere.com'  # ← CHANGE this
os.environ['OPENROUTER_API_KEY'] = 'your-openrouter-api-key'
os.environ['OPENROUTER_MODEL'] = 'openai/gpt-3.5-turbo'
os.environ['OPENROUTER_BASE_URL'] = 'https://openrouter.ai/api/v1'
os.environ['ADMIN_USERNAME'] = 'admin'
os.environ['ADMIN_PASSWORD'] = 'your-secure-admin-password'

# ──────────────────────────────────────────────
# 3. Activate your virtual environment
# ──────────────────────────────────────────────
activate_this = '/home/yourusername/aisolutions/venv/bin/activate_this.py'  # ← CHANGE this
with open(activate_this) as f:
    exec(f.read(), {'__file__': activate_this})

# ──────────────────────────────────────────────
# 4. Tell Django which settings module to use
# ──────────────────────────────────────────────
os.environ['DJANGO_SETTINGS_MODULE'] = 'config.settings'

# ──────────────────────────────────────────────
# 5. Start Django
# ──────────────────────────────────────────────
from django.core.wsgi import get_wsgi_application
application = get_wsgi_application()
CRITICAL: You must change all three yourusername placeholders to your actual PythonAnywhere username. Also set a strong SECRET_KEY and a secure ADMIN_PASSWORD.
16 Save the file WEB

Click the Save button (top-right of the editor), then close the editor tab.

6. Configure Static & Media Files

17 Why this is needed WEB

Django does not serve static files (CSS, JS, images) in production. PythonAnywhere has its own static file serving mechanism that intercepts requests to /static/ and /media/ before they reach Django. We need to tell it where those files are located.

18 Add static file mapping WEB

In the Web tab, scroll to the Static files section. You'll see two columns: URL and Directory. Add the following entries:

Entry 1 — Static files:

URLDirectory
/static//home/yourusername/aisolutions/staticfiles

Click Add to save this entry.

Entry 2 — Media files (uploads):

URLDirectory
/media//home/yourusername/aisolutions/media

Click Add to save this entry.

Replace yourusername with your actual PythonAnywhere username.

7. Configure CSRF Trusted Origins & CORS

19 Why this is needed WEB

Django's CSRF middleware checks the Origin or Referer header of incoming POST requests. If your domain is not listed in CSRF_TRUSTED_ORIGINS, POST requests (like login and form submissions) will fail with a 403 error.

20 Add trusted origins in the Web tab WEB

In the Web tab, find the CSRF trust section. It's usually below the Static files section. Add:

https://yourusername.pythonanywhere.com

Click Add.

21 Update CORS settings in settings.py FILES

This file should be edited locally on your computer using VS Code (or any code editor), then pushed to GitHub. PythonAnywhere will pull the updated code.

  1. On your local machine, open config/settings.py in VS Code.
  2. Find CORS_ALLOWED_ORIGINS and add your PythonAnywhere domain: 
    CORS_ALLOWED_ORIGINS = [
    'https://yourusername.pythonanywhere.com',
    'http://localhost:8000',
    'http://127.0.0.1:8000',
    'http://localhost:5500',
    'http://127.0.0.1:5500',
]
  3. Save the file, then commit and push to GitHub: 
    git add config/settings.py
git commit -m "add PythonAnywhere domain to CORS"
git push
  4. On PythonAnywhere, pull the update (see step 11 — "Deploy code changes").

10. Troubleshooting Common Issues

Problem: 500 Internal Server Error WEB

Solution: Check the error log. In the Web tab, under Logs, click Error log. Common causes:

  • Wrong path in WSGI file: The path variable points to the wrong directory. Verify it matches /home/yourusername/aisolutions.
  • Missing environment variables: The WSGI file doesn't have SECRET_KEY, ALLOWED_HOSTS, etc. set correctly.
  • Missing packages: The virtual environment doesn't have all required packages. Re-run pip install -r requirements.txt.
Problem: Static files not loading (CSS missing, no styling) WEB + CONSOLE

Solution:

  1. Verify the static file paths in the Static files section match your STATIC_ROOT.
  2. Re-run python manage.py collectstatic --noinput in the Bash console.
  3. Reload the web app after making changes.
Problem: Admin login fails (403 error) WEB

Solution: The CSRF trust origin may not be set correctly.

  1. In the Web tab, check the CSRF trust section has https://yourusername.pythonanywhere.com added.
  2. Also add it to CSRF_TRUSTED_ORIGINS in config/settings.py if not already there.
  3. Reload the web app.
Problem: CORS error in browser console FILES

Solution: Add your domain to CORS_ALLOWED_ORIGINS in config/settings.py as shown in step 21. Then reload.

Problem: "DisallowedHost" error WEB

Solution: Make sure ALLOWED_HOSTS in the WSGI file includes your exact PythonAnywhere domain (e.g. yourusername.pythonanywhere.com). Reload after fixing.

Note: Free tier limitations
  • Your web app will "go to sleep" if not accessed for a while. The first request after sleep may take 10-15 seconds to respond.
  • You can only have one web app on the free tier.
  • The SQLite database lives on the filesystem. On the free tier, the filesystem is ephemeral — if you upgrade or rebuild your web app, the database may reset. For persistent data, upgrade to a paid account or use PythonAnywhere's MySQL.

11. How to Update Your Site

Deploy code changes CONSOLE + WEB

Whenever you make changes to the code and push them to GitHub, follow these steps to update your live site:

cd ~/aisolutions
git pull
source venv/bin/activate
pip install -r requirements.txt      # Install any new packages
python manage.py migrate             # Apply any new database migrations
python manage.py collectstatic --noinput
deactivate

Then go to the Web tab and click Reload.

Update environment variables or WSGI config WEB

If you change the WSGI file or environment variables (e.g., update the OpenRouter API key), just go to the Web tab, edit the WSGI file, save, and click Reload. No need to run any terminal commands.
