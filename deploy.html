Complete guide for installing, running, and deploying the AI Solutions Django application.

2. Installation from ZIP File (Local Development)

Follow these steps to set up the project on your local machine from the ZIP archive or GitHub.

Step 1 Extract the ZIP File

Description: Unzip the project archive to a folder on your computer. This creates the project root directory containing all source files.

# Right-click the ZIP file → Extract All
# Choose a destination (e.g., C:\Users\YourName\Projects\)
# Ensure the extracted folder structure is:
aisolutions/
    manage.py
    requirements.txt
    config/
    core/
    chatbot/
    admin/
    css/
    js/
    photos/
    media/
    *.html

Step 2 Install Python 3.13

Description: Django 5.1.3 requires Python 3.10+. Python 3.13 is the recommended version. Download from the official Python website.

# Download from: https://www.python.org/downloads/
# During installation, check "Add Python to PATH"
# Verify installation:
py --version
# Output: Python 3.13.x

# If multiple Python versions are installed:
py -3.13 --version
Windows users: Use py -3.13 to explicitly launch Python 3.13. On Mac/Linux, use python3.13.

Step 3 Create a Virtual Environment

Description: A virtual environment isolates the project's Python packages from your system. This prevents version conflicts with other projects. The venv/ folder is created inside the project directory.

# Navigate to project folder
cd path\to\aisolutions

# Create virtual environment
py -3.13 -m venv venv

# Activate it
# Windows (Command Prompt/PowerShell):
venv\Scripts\activate

# Mac/Linux:
# source venv/bin/activate

# Your prompt should now show (venv)
Important: Always activate the virtual environment before running any Python/Django commands. The venv/ folder is listed in .gitignore and will not be committed to Git.

Step 4 Install Dependencies

Description: requirements.txt lists all Python packages the project needs. pip install -r requirements.txt downloads and installs every package and its dependencies automatically.

# Upgrade pip first (recommended)
pip install --upgrade pip

# Install all project dependencies
pip install -r requirements.txt

# Verify installation
pip list

Packages installed:

  • django 5.1.3 — core web framework
  • djangorestframework 3.15+ — REST API toolkit
  • django-cors-headers 4.3+ — cross-origin requests
  • requests 2.31+ — HTTP client for OpenRouter AI
  • gunicorn 22+ — production WSGI server (optional for local dev)

Step 5 Configure Environment Variables

Description: The application reads configuration from environment variables. Create a .env file (or set them in your shell) with your secrets. The .env file is listed in .gitignore and will NOT be committed to Git.

# Create .env in the project root (aisolutions/.env)
# Copy this template and fill in your values:

SECRET_KEY=django-insecure-your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=*
OPENROUTER_API_KEY=sk-or-v1-your-key-here
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-password
VariableRequiredDescription
SECRET_KEYYesDjango's cryptographic key. Used for sessions, CSRF tokens, etc. Generate a strong random key for production.
DEBUGYesTrue for development (shows detailed error pages). Must be False in production.
ALLOWED_HOSTSYesComma-separated domain names allowed to serve the site. Use * in development; set to your domain in production.
OPENROUTER_API_KEYNoYour OpenRouter API key for the AI chatbot. Without it, the chatbot uses keyword-based fallback responses.
ADMIN_USERNAMEYesUsername for the custom admin panel login at /admin/.
ADMIN_PASSWORDYesPassword for the custom admin panel login.

Step 6 Apply Database Migrations

Description: Migrations create the SQLite database tables based on the Django models (Event, User, Inquiry, DemoRequest, Feedback, PageVisit). Run this command every time models change.

# Create/update database tables
python manage.py migrate

# Expected output:
#   Applying core.0001_initial... OK
# This creates db.sqlite in the project root

Step 7 Collect Static Files

Description: Copies all CSS, JavaScript, and image files from their source directories (css/, js/, photos/) into a single staticfiles/ directory. This is required for production but also useful in development to verify the static file setup.

python manage.py collectstatic --noinput
# Creates staticfiles/ with all CSS, JS, and images

Step 8 Run the Development Server

Description: Django's built-in development server serves the application at http://127.0.0.1:8000. It auto-reloads when you change Python files. Open your browser and navigate to that address to see the site.

# Start the dev server
python manage.py runserver

# Open in browser:
#   http://127.0.0.1:8000  — Main website
#   http://127.0.0.1:8000/admin/  — Admin panel
#   http://127.0.0.1:8000/api/health/  — API health check

# Press Ctrl+C to stop the server
Note: The development server is for local testing only. Do not use it in production. Use Gunicorn or PythonAnywhere's WSGI runner instead.

3. Deployment to PythonAnywhere (Production)

These steps deploy the application to PythonAnywhere's free hosting tier at https://bishwasshrestha01.pythonanywhere.com.

3.1 Local Setup — Push Code to GitHub

Step 1 Initialize Git & Push to GitHub

Description: Initialize a Git repository in the project folder, stage all files, commit, and push to GitHub. PythonAnywhere will clone from this repository.

# VS Code Terminal (PowerShell) or any terminal in project root
cd C:\Users\YourName\path\to\aisolutions

# Initialize Git
git init

# Stage all files
git add .

# First commit
git commit -m "Initial commit - AI Solutions project"

# Link to GitHub (create the repo on GitHub first)
git remote add origin https://github.com/yourusername/aisolutions.git

# Push to GitHub
git push -u origin master
Important: Before pushing, ensure .gitignore exists and includes venv/, .env, db.sqlite, __pycache__/, and log files. This prevents pushing secrets and temporary files to GitHub.

3.2 PythonAnywhere — Initial Setup

Step 2 Create a PythonAnywhere Account & Web App

Description: Sign up at pythonanywhere.com (free tier is sufficient). Go to the Web tab and create a new web app. Choose Manual Configuration (not a template) and select Python 3.13.

# After creating the web app, note your site URL:
# https://yourusername.pythonanywhere.com

# Set these fields in the Web tab:
#   Source code:  /home/yourusername/aisolutions
#   Working dir:  /home/yourusername/aisolutions
#   Python:       3.13

Step 3 Clone the Repository on PythonAnywhere

Description: Open the PythonAnywhere Bash console (from the Consoles tab) and clone your GitHub repository. This downloads all your code to the server.

Tip: If you get permission errors, use the SSH clone URL instead, or configure a personal access token on GitHub.

Step 4 Create a Virtual Environment on PythonAnywhere

Description: Creates an isolated Python environment on the server and installs all project dependencies. This is identical to the local setup.

3.3 PythonAnywhere — Web App Configuration

Step 6 Configure the Virtual Environment Path

Description: Tell PythonAnywhere which virtual environment to use. In the Web tab, find the Virtualenv section and enter the path to your venv.

# In the Web tab, under "Virtualenv":
/home/yourusername/aisolutions/venv

# Click the refresh icon next to the field
# It should show: /home/yourusername/aisolutions/venv/bin/python
Troubleshooting: If you see "No virtualenv detected at this path", run these commands in the Bash console to recreate the venv:
cd ~/aisolutions
rm -rf venv
python3.13 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py collectstatic --noinput

Step 7 Configure the WSGI File

Description: The WSGI file tells PythonAnywhere how to run your Django application. It sets environment variables, activates the virtual environment, and starts Django.

Open the Files tab and edit /var/www/yourusername_pythonanywhere_com_wsgi.py. Replace the entire content with:

import os
import sys

# Project path
project_home = '/home/yourusername/aisolutions'
if project_home not in sys.path:
    sys.path.insert(0, project_home)

# Activate virtual environment
activate_this = project_home + '/venv/bin/activate_this.py'
if os.path.exists(activate_this):
    with open(activate_this) as f:
        exec(f.read(), {'__file__': activate_this})

# Environment variables
os.environ['DJANGO_SETTINGS_MODULE'] = 'config.settings'
os.environ['DEBUG'] = 'False'
os.environ['SECRET_KEY'] = 'your-production-secret-key'
os.environ['ALLOWED_HOSTS'] = 'yourusername.pythonanywhere.com'
os.environ['OPENROUTER_API_KEY'] = 'your-openrouter-api-key'
os.environ['ADMIN_USERNAME'] = 'admin'
os.environ['ADMIN_PASSWORD'] = 'your-admin-password'

# Start Django
from django.core.wsgi import get_wsgi_application
application = get_wsgi_application()
Security: Never share your WSGI file content publicly. The secrets (SECRET_KEY, API keys, admin password) must be unique and strong.

Step 8 Configure Static & Media File Mappings

Description: PythonAnywhere needs explicit mappings to serve static files (CSS, JS, images). These tell the web server where to find files when the browser requests /static/styles.css or /media/photo.jpg.

In the Web tab, scroll to Static files and add:

URL: /static/
Path: /home/yourusername/aisolutions/staticfiles/

URL: /media/
Path: /home/yourusername/aisolutions/media/

Step 9 Configure Security & Reload

Description: Enable HTTPS, set CSRF trust origins, and reload the web app to apply all changes.

# In the Web tab:

# Force HTTPS: ON (toggle at the top of the page)

# Under "Security"
# Add to CSRF trust origins:
https://yourusername.pythonanywhere.com

# Click the green "Reload" button
Reload is required every time you change the WSGI file, update Python packages, or change settings. It restarts your web application.

5. Common Issues & Solutions

ProblemCauseSolution
"No module named 'decouple'" Old code still uses python-decouple The project now uses os.environ.get() instead. Pull the latest code and ensure python-decouple is not in requirements.txt.
"No virtualenv detected at this path" Venv doesn't exist or path is wrong Recreate the venv: rm -rf venv && python3.13 -m venv venv && source venv/bin/activate && pip install -r requirements.txt
"DisallowedHost" error ALLOWED_HOSTS doesn't include the domain Add your PythonAnywhere domain to the ALLOWED_HOSTS environment variable in the WSGI file.
"CSRF verification failed" CSRF trust origin not set Add https://yourusername.pythonanywhere.com to the CSRF trust origins in the Web tab.
404 errors on CSS/JS/images Static files not collected or mapped Run python manage.py collectstatic --noinput and add the static file mapping in the Web tab.
500 "Internal Server Error" Various (missing env var, DB issue, etc.) Check the error log: cat /var/log/yourusername.pythonanywhere.com.error.log
Django welcome page shown WSGI file not configured Update the WSGI file with the correct DJANGO_SETTINGS_MODULE and project path. Ensure config.settings is the module name.
Chatbot not responding OpenRouter API key missing or invalid Set OPENROUTER_API_KEY in the WSGI file or .env. Without it, the chatbot uses keyword-based fallback responses.
